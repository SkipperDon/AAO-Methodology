Project Slash Commands — type in Claude Code chat
/project:session-start NEW

Load memory from prior sessions and orient for today's work. Reads MEMORY.md, PROJECT_CHECKLIST.md, and the last two SESSION_LOG entries. Run this first, every session.

/project:session-start

Also re-run after /compact to reload context lost in compaction.

/project:session-close SESSION

End-of-session governance sequence. Updates SESSION_LOG, PROJECT_CHECKLIST, DEPLOYMENT_INDEX, CHANGELOG, MEMORY, and BUILD_CHECKLIST. Presents final summary. All steps mandatory.

/project:session-close

Never skip. Close writes — start reads. Together they form the memory loop.

/project:methodology-check AUDIT

Mid-session self-audit. Verifies risk classification, scope boundaries, and AAO compliance checklist. Run when a session is going long or scope starts to drift.

/project:methodology-check
/project:bug-fix TDD

TDD-enforced bug fix workflow. Write a failing test that reproduces the bug first. The fix is only written after the failing test exists. A test written after the fix is documentation, not a test.

/project:bug-fix
Operator Chat Commands — backup management
/aao-backup-status BACKUP

List every backup file in .aao-backups/, grouped by session. Shows original file, backup age, and purge eligibility. Read-only — never deletes anything.

/aao-backup-status

Safe to run at any time. Use before purge to review what exists.

/aao-backup-purge BACKUP

List all purge-eligible backups (older than 3 most recent per file), wait for explicit confirmation, then delete. Reports exactly what was removed and what remains.

/aao-backup-purge

Always lists before deleting. Requires unambiguous confirmation. Never purges the current session's folder.

Sprint Mode — activation phrases

Sprint Mode

Bounded task execution. Claude Code executes only named tasks, stops after each one, and waits for explicit approval before proceeding. Overrides all Autonomous Operation defaults. Defined structurally in CLAUDE.md — not a verbal convention.

Risk Classification — applied before every action
Level Covers Required Behaviour Confirmation
NONE Read-only — reading files, checking status, searching Execute without stating. No confirmation required. —
LOW State-changing, reversible — editing files, creating docs State what you are about to do. In Sprint Mode: wait for acknowledgment. Implicit
MEDIUM Meaningful config changes — modifying deployment scripts State the action, impact, and rollback path. Wait for approval. Explicit
HIGH Irreversible or externally visible — git push, rm -rf, external APIs STOP. State action, impact, rollback path. Require named confirmation phrase. Named phrase
