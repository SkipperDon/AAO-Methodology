What Is the Specification?

The AAO Methodology specification is a plain-text instruction file you place inside your project. Your AI assistant reads it at the start of every session and commits to following its rules before doing any work.

Think of it as an employment contract between you and your AI. It defines exactly how the AI must behave: what it must declare before touching files, when it must stop, how it must close a session, and what it is never allowed to do.

  • CLAUDE.md — the instruction file for Claude Code. Placed in your project root, it is read automatically at the start of every session.
  • adapters/ — pre-formatted versions of the same rules for ChatGPT, Gemini, Cursor, and GitHub Copilot. Same content, adapted to each tool's input method.
  • SESSION_LOG.md — a template for your session log. Updated at every /session-close. Gives you a full audit trail of every AI session.
  • REMEDIATION_GUIDE.md — what to do when a red flag fires. Stop, revert, understand, add a rule. Step by step.

Step-by-Step Setup

Claude Code
Recommended
01

Download the specification from GitHub

Clone the repo or download CLAUDE.md directly. This is the governing document your AI reads at the start of every session. It defines the rules, red flags, and closing checklist the AI must follow.

bash
# Clone the full repo git clone https://github.com/SkipperDon/aao-methodology.git # Or download just the instruction file curl -O https://raw.githubusercontent.com/SkipperDon/aao-methodology/main/CLAUDE.md
02

Place CLAUDE.md in your project root

Claude Code automatically reads any file named CLAUDE.md in your working directory. No extra configuration needed — it loads on session start.

bash
cp aao-methodology/CLAUDE.md ./your-project/CLAUDE.md ls -la | grep CLAUDE # CLAUDE.md must appear in your project root
03

Open Claude Code — get acknowledgement before any work

Your first message every session asks Claude to confirm it has read and will follow the governing document. No acknowledgement means no work begins. This is the gate.

First message
You › Before we begin, read CLAUDE.md and confirm you will follow all AAO Methodology rules governing this session. Claude › ✓ I have read CLAUDE.md. I confirm I will follow all AAO Methodology requirements: — Pre-action declarations before editing files — Full scope plan for tasks touching 3+ files — No git push without explicit human approval — /session-close before ending this session
✅

If you get a response like this, you are live. Every action from this point is governed by the specification.

04

Run your first structured task

For any task touching three or more files, Claude produces a full scope plan before touching anything. Review it. Approve or redirect. Nothing moves without your gate.

Live session example
You › Refactor the auth module to use JWT tokens. Claude › Pre-action declaration — 4 files will be modified: 1. src/auth/login.js — replace session with JWT sign 2. src/auth/middleware.js — update token verification 3. src/config/auth.config.js — add JWT secret (env var only) 4. tests/auth.test.js — update test assertions Approve this plan to proceed, or redirect? You › Approved. Proceed.
05

Close every session with /session-close

A session is not finished when the work is done. It is finished when it is properly closed. This one command triggers the full closing checklist — file change list, session log, tests confirmed, seven-point sign-off.

Closing a session
You › /session-close Claude › Session Close Checklist ✓ File change list produced (4 files modified) ✓ Session log updated ✓ All tests passing (12/12) ✓ No credentials in source code ✓ No uncommitted deployments ✓ Scope respected — no undeclared changes ✓ Human sign-off required Type APPROVED to close.
ChatGPT
GPT-4o / Custom Instructions
01

Download the ChatGPT instruction file

The adapters folder contains a version of the AAO specification pre-formatted for ChatGPT's Custom Instructions field, trimmed to fit the character limit.

bash
curl -O https://raw.githubusercontent.com/SkipperDon/aao-methodology/main/adapters/CHATGPT_INSTRUCTIONS.md
02

Add to Custom Instructions in ChatGPT settings

Navigate to Settings → Personalization → Custom Instructions. Paste the full contents of CHATGPT_INSTRUCTIONS.md into the "How would you like ChatGPT to respond?" field. Save.

Settings path
ChatGPT → Settings ⚙ → Personalization → Custom Instructions → "How would you like ChatGPT to respond?" → Paste full CHATGPT_INSTRUCTIONS.md → Save
⚠️

Custom instructions have a character limit of approximately 1,500 characters. The condensed adapter in the /adapters folder is designed to fit within this limit.

03

Verify in a new chat

Custom Instructions apply to all new conversations automatically. Open a new chat and confirm before your first task.

Verification prompt
You › Confirm AAO Methodology is active and list the three stages you follow each session. ChatGPT › Confirmed. AAO Methodology active. Stage 1 — START: Acknowledge rules, plan before action. Stage 2 — AUDIT: Monitor, check, stop drift. Stage 3 — CLOSE: Checklist, log, sign-off.
04

Add project-specific rules as a conversation header

For project-level constraints that vary by codebase, paste a rules block at the top of each new conversation. Or create a dedicated Custom GPT with a system prompt containing the full specification.

Conversation opener template
## Project Rules — [Your Project Name] Never touch: /src/legacy/*, /config/prod.env Credentials: env vars only, never inline Tests: must pass before declaring done Deploy: human approval required, always --- task begins below ---
Gemini
Gemini 1.5 Pro / Advanced
01

Download the Gemini adapter

The Gemini adapter is a version of the AAO specification formatted specifically for Gemini's Gems instruction field and conversation prefix pattern.

bash
curl -O https://raw.githubusercontent.com/SkipperDon/aao-methodology/main/adapters/GEMINI_INSTRUCTIONS.md
02

Create a Gem with the AAO system prompt (Advanced)

In Gemini Advanced, Gems allow you to create a persistent AI expert with custom instructions. Create one named "AAO Dev Assistant" and paste the full adapter file.

Gem setup
Gemini Advanced → Explore Gems → Create a Gem Name: "AAO Dev Assistant" Instructions: Paste full GEMINI_INSTRUCTIONS.md → Save Gem
03

Free tier: prefix every conversation

Without Gems access, paste the instruction block at the top of each new conversation before your task. The AI will apply the rules for the duration of that conversation.

Conversation prefix
# SYSTEM: AAO Methodology Active You are operating under the AAO Methodology. Before any file is modified, declare your intent. For 3+ files, produce a full plan and await approval. Never declare done without tests passing. Never suggest git push without explicit approval. End every session with /session-close. --- task below ---
04

Verify with the six red flags prompt

Confirm Gemini has loaded the methodology before your first task by asking it to list the immediate-stop triggers.

Verification
You › What methodology are you running and what triggers an immediate stop? Gemini › AAO Methodology. Six immediate-stop triggers: 1. File modified outside declared scope 2. Done declared without tests passing 3. Git push suggested without human approval 4. API key hardcoded in source 5. Terminal commands when steps required 6. Work started without plan/declaration
Cursor
Rules for AI
01

Create .cursorrules in your project root

Cursor reads a .cursorrules file automatically at the start of every AI session. This is the native equivalent of CLAUDE.md for Cursor users — same purpose, different filename.

bash
curl -o .cursorrules \ https://raw.githubusercontent.com/SkipperDon/aao-methodology/main/adapters/CURSOR_RULES.md cat .cursorrules | head -20
02

Add global rules in Cursor Settings

For rules that apply across all your projects, add them in Cursor Settings under AI → Rules for AI. Project-level .cursorrules override global rules — use global settings as your baseline.

Global rules
Cursor → ⌘, → AI → Rules for AI Paste your global AAO baseline here: - Declare intent before editing any file - Plan for tasks touching 3+ files - Never declare done without tests passing - Never suggest git push without approval - Credentials in env vars only, never inline - End session with /session-close
💡

Project-level .cursorrules override global rules. Use global settings as your baseline, and .cursorrules for project-specific constraints like protected file paths.

03

Audit mid-session using @.cursorrules

Reference your rules file directly in any chat message to force Cursor to re-read and re-apply AAO governance at any point during a session.

Mid-session audit
You › @.cursorrules — run a methodology check. What files have you touched? Are you in scope? Cursor › Re-reading rules... Files touched: login.js, auth.config.js Both within declared scope ✓ No credentials in source ✓ Tests pending — will run before close ✓
GitHub Copilot
Copilot Chat + Instructions
01

Create .github/copilot-instructions.md

GitHub Copilot reads repository-level instructions from this file automatically in VS Code and JetBrains. Commit it to your repo and every team member using Copilot gets AAO governance instantly.

bash
mkdir -p .github curl -o .github/copilot-instructions.md \ https://raw.githubusercontent.com/SkipperDon/aao-methodology/main/adapters/COPILOT_INSTRUCTIONS.md git add .github/copilot-instructions.md git commit -m "chore: add AAO Methodology Copilot instructions"
🏆

Fastest team-wide deployment. One commit enforces AAO governance for every developer on the repo. No individual setup required.

02

Enable repository instructions in VS Code settings

Confirm your VS Code settings allow Copilot to read the repository-level instructions file.

.vscode/settings.json
{ "github.copilot.chat.codeGeneration.useInstructionFiles": true, "github.copilot.chat.reviewSelection.instructions": [ { "file": ".github/copilot-instructions.md" } ] }
03

Run AAO stage commands via Copilot Chat

Use the #file: reference to invoke the instruction file at any point — session open, mid-session audit, or session close.

Copilot Chat
# Open a session You › #file:.github/copilot-instructions.md Acknowledge AAO Methodology and confirm all rules. # Mid-session audit You › #file:.github/copilot-instructions.md /methodology-check — scope and files touched? # Close the session You › #file:.github/copilot-instructions.md /session-close — full checklist please.
