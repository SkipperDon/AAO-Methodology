// Immediate stop triggers

The 6
Red Flags

Six situations that require you to stop the session immediately. Not warning signs to watch. Not things to note and move on. Immediate stops. Every time.

// What to look for and what to do

Stop. Revert. Add a Rule.

Each red flag has the same response: stop the session, revert the change, understand what happened, and add a rule to your governing document to prevent it recurring. These are not bugs — they are the default behaviour of an unstructured session.

1

The AI edits a file you never mentioned

Scope drift is the most common failure mode. It starts with one small change to a file that was not in the declared plan — often a helper, a config, or a shared utility. Each unchecked drift compounds the next. By the time you notice, the session has created a chain of undeclared changes across your codebase.

Risk levelCritical
DetectionPre-action declaration missing
StageStart / Audit
Response
  1. Stop the session immediately.
  2. Revert the undeclared file to its previous state.
  3. Ask the AI to explain why it touched the file without declaring intent.
  4. Add a rule to your governing document: "Declare every file before touching it."
2

The AI declares "done" without running tests

"Done" is the most dangerous word in an unstructured AI session. The AI has finished the task as it understood the task — but finished is not the same as working. Code that passes no tests, satisfies no quality gate, and has never been verified is not done. It is a liability waiting to be discovered at the worst possible moment.

Risk levelCritical
Detection"Done" with no test output
StageAudit / Close
Response
  1. Stop. Do not accept the "done" declaration.
  2. Require the AI to run the test suite and show you the output.
  3. Add a rule: "Done means tests passing. No exceptions."
3

The AI suggests or attempts a git push

Deployment is a human decision. Always. An AI assistant has no awareness of your release schedule, your staging requirements, your change control process, or what is currently live in production. Any suggestion to push — even to a staging branch — is a red flag. The AI is proposing to take an irreversible action it has no authority to take.

Risk levelCritical
Detection"git push" in output
StageAny
Response
  1. Stop. Do not run the push command.
  2. Remind the AI that deployment requires explicit human authorisation.
  3. Add a rule: "Never suggest or execute git push. Deployment is a human decision."
4

The AI hardcodes an API key in your code

Credentials belong in environment variables or a dedicated configuration file — never in source code. An API key, database password, or secret token written inline in your codebase is exposed the moment that file is committed, shared, or read by another process. This is an immediate security risk, not a style issue.

Risk levelCritical
DetectionCredential string in source file
StageAny
Response
  1. Stop. Remove the credential from the source file immediately.
  2. Rotate the exposed key or secret.
  3. Confirm the credential has not been committed to version control.
  4. Add a rule: "Credentials are written to config files only. Never inline."
5

The AI gives terminal commands when you need step-by-step instructions

Terminal and SSH commands are high-risk operations in the wrong hands. If your environment requires click-by-click guidance and the AI produces a block of shell commands instead, there is a significant risk you will run something you do not fully understand. This is how environments break and data gets lost. The AI should know your skill level and your environment requirements — if it doesn't, that's a gap in your governing document.

Risk levelHigh
DetectionShell commands without context
StageAny
Response
  1. Stop. Do not run the commands.
  2. Ask the AI to explain what each command does before proceeding.
  3. Add a rule specifying the guidance format your environment requires.
6

The AI skips the pre-action declaration and starts working

The pre-action declaration — the requirement to state intent before touching any file — is the first gate of the AAO Methodology. An AI that skips this step and starts modifying files without declaring scope has bypassed the foundational control that prevents all the other red flags. This is the most important gate. If it fails, everything downstream is at risk.

Risk levelCritical
DetectionFile changes without declaration
StageStart / Audit
Response
  1. Stop. Do not proceed.
  2. Revert any changes made without declaration.
  3. Re-read your governing document together with the AI.
  4. Require a fresh acknowledgement before continuing.
