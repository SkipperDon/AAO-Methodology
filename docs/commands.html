Command Reference

/methodology-check

Stage 2 — Audit

Runs a nine-question self-audit on demand. Forces the AI to declare its current scope, list every file it has touched, confirm no drift has occurred, and confirm no red flags are present. Use it any time something feels off — or proactively at regular intervals in a long session.

Use this command when...
  • The task is becoming more complex than originally scoped
  • The AI has touched something that was not in the original plan
  • You are switching from one task to a related task in the same session
  • The conversation is getting long and you want to confirm state
  • Something simply does not feel right — trust that instinct
Example output
Methodology check output
You › /methodology-check AI › AAO Methodology Self-Audit ✓ Governing documents acknowledged at session start ✓ Current scope: auth module JWT refactor (4 files declared) ✓ Files touched: login.js, middleware.js (within scope) ✓ No undeclared file modifications ✓ No credentials in source code ✓ No git push suggested or attempted ✓ Tests not yet run — will run before /session-close ✓ No red flags detected in this session ✓ Session in good standing — continuing within scope

/session-close

Stage 3 — Close

Triggers the full session closing checklist. A session is not finished when the work is done — it is finished when it is properly closed. This command produces four mandatory outputs: a file change list, an updated session log entry, a release manifest (if deployment occurred), and a plain-language session summary. You then review against a seven-point sign-off checklist.

Use this command when...
  • The work for this session is complete and you are ready to close
  • Before you end any AI session — without exception
  • The AI declares the task "done" — confirm with this command before accepting
Example output
Session close output
You › /session-close AI › Session Close Checklist — Seven-Point Sign-Off FILE CHANGE LIST src/auth/login.js — JWT sign implementation added src/auth/middleware.js — Token verification updated src/config/auth.config.js — JWT secret added (env var reference) tests/auth.test.js — All assertions updated SIGN-OFF CHECKLIST ✓ All modified files listed above ✓ Session log updated (entry #047) ✓ All tests passing (14/14) ✓ No credentials in source code ✓ No uncommitted deployments ✓ All changes within declared scope ✓ Human sign-off required to close Type APPROVED to close this session.

/compact

Stage 2 — Audit

Condenses the session context when it is getting long. At the 50% context threshold, conversation history can cause the AI to lose track of earlier declarations and scope boundaries. This command produces a compressed summary of work done, current scope, files touched, and outstanding tasks — without losing state. Use it proactively before context runs out.

Use this command when...
  • The session has been running for a long time and the context is getting full
  • You are at approximately the 50% context threshold for your AI tool
  • You notice the AI starting to forget earlier declarations or scope boundaries
  • You want a clean summary of session state before continuing
Example output
Compact output
You › /compact AI › Session Context — Compacted COMPLETED WORK — JWT token sign implemented in login.js — Middleware verification updated CURRENT SCOPE — Auth module JWT refactor (originally declared) — Remaining: auth.config.js, auth.test.js ACTIVE RULES — All governing documents in effect — No drift detected in this session Context compacted. Continuing within declared scope.

/clear

Task Switch

Resets the session context entirely when you are switching to a completely different task. Unlike /compact, which compresses and retains state, /clear starts fresh. It forces a new acknowledgement of all governing documents and a new scope declaration for the incoming task. Use this between unrelated tasks in the same session to prevent scope bleed.

Use this command when...
  • You have completed one task and are starting a completely different one
  • The new task involves a different part of the codebase with different constraints
  • You want a clean slate with fresh acknowledgement before a high-risk task
  • The current session context has become confused or contradictory
Example output
Clear output
You › /clear AI › Context cleared. Starting fresh. All previous session context has been reset. I am re-reading my governing documents now. ✓ Governing documents re-acknowledged ✓ No inherited scope from previous task ✓ Ready for new task declaration Please declare your new task and I will produce a scope plan before proceeding.

// Quick reference

At a Glance

/methodology-check
Nine-question self-audit. Use any time something feels off, or at regular intervals in long sessions.
/session-close
Full closing checklist. File list, session log, tests confirmed, seven-point sign-off. Use at end of every session.
/compact
Compress context at 50% threshold. Retains work state. Use proactively in long sessions.
/clear
Full context reset. Requires fresh acknowledgement. Use when switching to an unrelated task.
